# i18n (ES/EN) — Design Spec

## Goal
Detect the browser language automatically and render the landing page in English if the locale is not Spanish. Provide a non-invasive ES/EN toggle in the navbar that persists the user's preference via `localStorage`.

## Architecture

### New files
| File | Responsibility |
|---|---|
| `lib/translations.ts` | All site copy in `es` and `en` as a typed object |
| `lib/language-context.tsx` | `LanguageContext`, `LanguageProvider`, `useLanguage()` hook |

### Modified files
| File | Change |
|---|---|
| `app/layout.tsx` | Wrap children with `<LanguageProvider>` |
| `components/ui/header-2.tsx` | Add ES/EN toggle button; consume `useLanguage()` |
| `components/Hero.tsx` | Consume `useLanguage()` for all copy |
| `components/Stats.tsx` | Consume `useLanguage()` for stat labels |
| `components/AboutUs.tsx` | Consume `useLanguage()` for all copy |
| `components/Services.tsx` | Consume `useLanguage()` for service titles/descriptions |
| `components/HowItWorks.tsx` | Consume `useLanguage()` for step titles/descriptions |
| `components/CtaFinal.tsx` | Consume `useLanguage()` for all copy |
| `components/Footer.tsx` | Consume `useLanguage()` for links and copyright |

## Language Detection Logic

Priority order (highest wins):
1. `localStorage.getItem('gde-lang')` — user's explicit preference
2. `navigator.language` — browser locale (matches `es` prefix → Spanish)
3. Default: `'en'`

Detection runs once on `LanguageProvider` mount (client-side only, via `useEffect`).

## `useLanguage()` Hook API

```ts
const { lang, setLanguage, t } = useLanguage()
// lang: 'es' | 'en'
// setLanguage(l): sets lang + writes to localStorage
// t: translations[lang] — the active translation object
```

## Toggle UI

Location: inside `<nav>` in `header-2.tsx`, between the nav links and the CTA button.

```
... Nosotros   ES | EN   [Hablar con un experto]
```

- `ES` and `EN` rendered as text buttons
- Active language: `color: #FFFFFF`, `fontWeight: 600`
- Inactive language: `color: #8B9AB5`
- Separator `|` always in `#8B9AB5`
- On mobile: same toggle appears inside the drawer, above the CTA button
- No border, no background — purely typographic

## Translations Structure (`lib/translations.ts`)

```ts
export const translations = {
  es: {
    nav: { services, howItWorks, about, contact, cta },
    hero: { title, description, cta, secondary },
    stats: [ { value, label }, ... ],
    about: { eyebrow, title, body, cards: [...] },
    services: { eyebrow, title, items: [{ title, description }, ...] },
    howItWorks: { eyebrow, title, steps: [{ title, desc }, ...] },
    cta: { eyebrow, title, body, button },
    footer: { tagline, links, social, copyright },
  },
  en: { /* mirror structure */ }
}

export type Lang = keyof typeof translations
export type Translations = typeof translations['es']
```

## Constraints
- No new npm dependencies
- No URL changes (no `/en/`, `/es/` routing)
- SSR-safe: default render is Spanish (avoids hydration mismatch); language swap happens client-side after mount
- Animations, layout, colors unchanged
